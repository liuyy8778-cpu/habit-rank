## 0. 一句話定位

一個幫**家庭一起養成自律習慣**的手機 web app,核心是**訓練小孩的「自我控制肌肉」**,
而不是用外力強迫。以「段位 / 自律值」遊戲化,小孩與家長雙視角。

---

## 1. 核心理念(最重要,請優先評估這段)

**我們要訓練的是「自律」,不是「服從」。**

| | 他律(我們刻意避免) | 自律(我們要的) |
|--|------|------|
| 例子 | Family Link 時間到自動鎖死手機 | 手機還能用,小孩**自己選擇**放下 |
| 訓練到的 | 服從(外力一消失就打回原形) | 自制(肌肉留下來) |

**關鍵信念:自律的肌肉,只有在「有得選、卻選擇放下」的時候才會長。**
強制鎖機把「選擇」拿掉了,等於也拿掉了訓練的機會。

由此推出的核心迴圈:

```
有得選 → 自願放下 → 打卡留證 → 換取信任 → 最終「畢業」到純自律
```

「驗證」是訓練輪,**最終目標是拆掉它**——小孩努力的方向,就是「讓爸媽再也不用檢查我」。
如果驗證永遠都在,它也會變成另一種他律。所以**「畢業 / 免檢查」本身應該是被慶祝的里程碑。**

## 2. 目前已經做好的功能(現況)

- **雙視角**:小孩 5 分頁(今日 / 任務 / 段位 / 獎勵 / 紀錄)、家長 3 分頁(待確認 / 週報 / 任務·獎勵)。
- **三種貨幣**:
  - 🪙 金幣:花掉,兌換獎勵。
  - ⚡ 自律值 XP:只累積,決定段位。
  - 🔥 連續(streak):每日達成累積,可保護。
- **自律任務庫**:21 項任務,跨 6 大領域(螢幕 / 作息 / 學習 / 身體 / 責任 / 情緒·誠實)。
  每項標領域、金幣、XP、圖示;分「關鍵習慣」與「每日任務」。
- **段位系統**:見習 → 銅 → 銀 → 金 → 鑽石 → 傳說(XP 門檻 0/300/800/1800/3500/6000,傳說約半年)。
- **信任升級制**:段位越高,驗證越鬆(見習=每次確認 → 銀/金=隨機抽查 → 鑽石/傳說=完全自主)。
- **衝動延遲鈕**:想滑手機時先進入 10 分鐘冷靜,訓練「先暫停」,累計「已忍住 N 次」。
- **誠實值**:「誠實回報(含誠實承認沒做到)」的任務會餵這個值——鼓勵對自己誠實。
- **今天模式 + 日期感知**:
  - 在家 / 上學日 / 出門日三種模式,依日期自動預設(週末在家、暑假平日在家、開學平日上學日),可手動切換。
  - 出門日只顯示「到哪都能做」的任務,連續不中斷。
  - 每日自動換日重置、連續自動累積/中斷、出門日保護連續。
- **家長任務管理**:家長可開關任務庫中哪些對小孩生效。
- **經濟平衡**:初始全 0;完美一天約 50 幣 / 40 XP;獎勵定價 250~1200,最便宜也要存約 5 天(刻意「不太簡單」)。

### 技術現況
- 原本是 Claude Design 匯出的「打包好的」單一 HTML;已拆成**可編輯專案**:
  `src/app.js`(邏輯)、`src/template.html`(版面)、`build.py`(打包回 index.html)、`vendor/`(runtime + 字型)。
- 已把 React / ReactDOM / Babel 從 unpkg CDN 改為**自架 `/cdn`**(移除外部依賴,解決隨機空白)。
- **目前無帳號、無後端**,資料全存在各裝置的 `localStorage`(所以手機和電腦各是一份、不同步)。

## 3. 最近一輪討論「定案」的方向

### 3.1 跨裝置同步(要加後端)
- 用 **Firebase**(免費額度):Auth 登入 + Firestore 資料同步 + Storage 照片。
- **一個「家庭空間」**,多裝置加入,每台選角色(家長 / 小孩)。
- 登入:**家長 Google 一鍵**;**小孩裝置用「家庭代碼」加入**(不需小孩有 Google 帳號)。
- 解決:手機 = 電腦同一份進度、換機不遺失、家長/小孩即時互通。

### 3.2 交機認定(核心自律機制)
- **自願交機打卡**:小孩自己把手機放進充電座 → 按「我交機了」→(舊手機拍一張充電區照片當證明)。
- 照片是「**驕傲的打卡章**」,不是監視——同一台相機,「我打卡」和「我被監視」是相反心理,設計上必須是前者。
- **判定交給家長一鍵確認,不用 AI 辨識**(AI 判斷手機在不在座上不可靠、易誤判引爭執;家長一眼一鍵,還能通融)。
- 攝影機用**一支舊手機**對著充電區(不走市售監視器,整合最順)。
- **隱私**:照片確認完自動刪,最多保留 3 天(不無限存小孩空間照片)。
- 與信任升級制綁定:見習期拍照 → 做到、贏得信任 → **畢業到只需自己回報、免拍照**。

### 3.3 時間進度條
- 選定 **「全天時間軸」**:一條橫軸標出「可用時段(綠)/ 交機時段(灰)」+ 現在位置游標。
- 用意:讓小孩看到**一天的節奏全貌**(是教育,不是催促倒數)。
- 等於順手把「交機時段表」做出來。

### 3.4 對 Family Link 的立場
- 只當**安全網**(真的失控時的最後防線),**不是主力**,而且要隨信任逐步拿掉。
- 理由見第 1 節:它訓練服從,不訓練自律。

## 4. 還沒決定的開放問題(想聽意見)

1. **「畢業(免檢查 / 免拍照)」該綁哪個段位?** 還是做成獨立的「信任里程碑」?
   這會影響整個信任升級制怎麼設計。
2. **交機時段表:平日 / 假日要兩套嗎?** 上學日小孩在校,原本的三個使用窗口(11:00 / 14:00 / 17:00)用不到;
   假日才適用。傾向平日一套、假日一套,接「今天模式」。具體時間數字未定。
3. **離線**:小孩裝置沒網路時是否照樣能打卡、連上再同步?(傾向:可以,localStorage 當快取。)
4. **提醒**:22:00 是否要溫和推播提醒交機?(注意 iPhone 網頁推播限制較多。)

5. **家長參與**:目前偏「家長管小孩」,我們想加「家長也一起自律 + 家庭共同目標」讓它變「一家人同隊」。
